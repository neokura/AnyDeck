# Support

## Before Opening an Issue

Please include as much of this context as you can:

- handheld model
- vendor
- SteamOS version
- whether the device is part of validated support or experimental support
- which view or toggle failed
- what you expected
- what actually happened
- whether the backend was running with Decky root mode

## Useful Checks

- Confirm you are on SteamOS `3.8+`.
- Confirm the device is not a Steam Deck.
- Confirm the feature is shown as available in the UI.
- Reproduce once, then capture the `Information` view details and debug log snapshot.

## What Counts as a Good Report

- exact steps
- exact device
- exact SteamOS build
- logs or screenshots
- whether the issue is always reproducible

## What Support Does Not Mean

- broad distro support outside SteamOS
- guaranteed parity across every AMD x86 handheld
- hidden unsafe tuning defaults

If you are reporting a feature gap on a new handheld family, that is still useful. The most helpful reports are the ones that map what the hardware and SteamOS interfaces actually expose.
